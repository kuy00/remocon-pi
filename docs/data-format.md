# 데이터 형식 & 프로토콜 학습

특정 에어컨에 맞춘 고정 해석은 두지 않는다. **바이트의 의미는 `ir_learn.py`가
수집 데이터에서 자동으로 학습**한다. 이 문서는 공통 포맷과 학습 결과 구조를 설명한다.

## 원시 펄스(segs) 형식

모든 신호는 `[level, 마이크로초]` 쌍의 리스트(`segs`)로 저장한다.

- `level 0` = mark (38kHz 캐리어 ON) — 예: 헤더 `[0, 6800]`
- `level 1` = space (캐리어 OFF) — 예: 헤더 `[1, 3330]`
- 송신 시 이 극성 그대로 캐리어 ON/OFF로 변환한다(`ir_send.py`).

> 수신기(KY-022 등)는 보통 active-low지만, 녹화·재생에 같은 규칙을 쓰므로 결과는 일치한다.

## 수집 파일 — `dataset/{라벨}.json`

`ir_collect.py`가 설정 하나당 한 파일로 저장한다.

```jsonc
{
  "params": { "mode": "냉방", "temp": 21, "power": "on" },  // 라벨 = 값들을 '_'로 결합
  "frame_gap_us": 30000,
  "glitch_us": 150,
  "n_repeats": 8,
  "confidence": 1.0,          // 반복본 합의 일치율 (1.0 = 8회 모두 동일)
  "repeats": [ segs, segs, ... ]   // N회 반복 raw 펄스 (다수결/신뢰도 계산용)
}
```

`params`의 키는 `sweep.json`의 축이며 리모컨마다 자유롭게 정의한다(하드코딩 아님).

## 디코딩 (segs → 바이트), 타이밍 무관

`ir_codec.py`가 담당한다.

- **프레임 구분**: 긴 무신호 갭(기본 20000µs)으로만 나눈다(헤더 타이밍 하드코딩 없음).
- **비트 판정**: 각 캡처의 space 길이들을 짧은무리/긴무리로 **스스로 군집화**해 0/1 경계를 정한다.
- **바이트 조립**: LSB-first, 8비트 단위.
- 한 전송은 여러 프레임(F1, F2 ...)으로 나뉠 수 있고, 프레임 안은 바이트 배열이 된다.

## 학습 모델 — `model.json`

`ir_learn.py`가 `dataset/`를 분석해 만든다. 프레임별 바이트마다 종류를 분류한다.

| 종류 | 의미 | 예측 |
|------|------|------|
| `const` | 모든 샘플에서 고정 (주소/상수) | 고정값 |
| `field` | 특정 파라미터(또는 조합)로 결정 | `linear`(선형)면 미수집 값도 계산, `lookup`이면 표 |
| `checksum` | 다른 바이트들(**프레임 교차 포함**)의 합/XOR(+보정상수) | 합성 시 계산 |
| `complex` | 규칙 미해독 | 합성 불가 → 해당 프레임은 replay 필요 |

학습 절차:
1. 같은 설정의 반복본을 **다수결**로 합쳐 노이즈 제거(신뢰도 낮으면 경고)
2. **차분 분석** — 한 파라미터만 바뀔 때 함께 변하는 바이트 = 그 파라미터의 필드
3. 항상 고정 = `const`, 단일/조합 파라미터로 설명되면 `field`
4. 나머지는 체크섬 후보 알고리즘을 탐색, 맞으면 `checksum`, 아니면 `complex`
   - 전체 프레임 바이트를 순서대로 평탄화해 **프레임 교차 연속 구간**까지 payload 후보로 본다
     (F2 끝바이트가 F1 전체의 체크섬인 경우 등). 보정상수는 자동 추정.
   - `range`는 `[frame, byte]` 쌍 리스트. 노이즈 오탐을 줄이려 **고신뢰(≥0.9) 샘플 우선** 탐색.

> 미해독(`complex`) 바이트가 0이면 그 프레임은 완전 합성 가능. 남으면 해당 프레임은
> 가장 가까운 수집본을 replay 한다. 체크섬 해독은 **깨끗한 데이터(높은 신뢰도)**가 전제다.
